# Getting Started

> **Disclaimer:** This document was generated by an AI and may contain inaccuracies. If you find any errors, please open an issue on the [Nekomata GitHub repository](https://github.com/AlephAlpha/Nekomata).

This page is a practical introduction to running Nekomata locally. It covers:

- installing the compiler and build tools
- building or installing Nekomata
- running programs from the command line
- using the REPL
- choosing an output mode

## Requirements

Nekomata is written in [Haskell](https://www.haskell.org/). To build it, you need:

- [GHC](https://www.haskell.org/ghc/) 9.6.7 or later
- [Cabal](https://www.haskell.org/cabal/) 3.0 or later

The easiest way to get both is [GHCup](https://www.haskell.org/ghcup/).

Package-manager versions on some Linux distributions may be too old.

## Build and Install

From the project root, build Nekomata with:

```bash
cabal build
```

If you want to install the executable into your system environment, run:

```bash
cabal install
```

If you do not want to install it, you can run it directly through Cabal:

```bash
cabal run Nekomata
```

If you see commands below that start with `Nekomata`, you can replace them with:

```bash
cabal run Nekomata --
```

## Quick Start

Run a program directly from the command line:

```bash
Nekomata -c '"Hello, World!"'
```

Run a program with input:

```bash
Nekomata -c '+' -i '1 2'
```

Start the REPL:

```bash
Nekomata
```

## Running Programs

### Run code from the command line

Use `-c` to pass source code directly:

```bash
Nekomata -c '"Hello, World!"'
```

### Run code from a file

Nekomata can read source files in two encodings:

- `-u FILE` for UTF-8 source
- `-f FILE` for Nekomata's custom code page

Example:

```bash
Nekomata -u hello.neko
```

### Pass input on the command line

Use `-i` to provide stack input as a space-separated string:

```bash
Nekomata -c '+' -i '1 2'
```

Input can also be separated by commas:

```bash
Nekomata -c '+' -i '1,2'
```

Input supports comments starting with `->`:

```bash
Nekomata -c '+' -i '1 2 -> expected result is 3'
```

Everything after `->` is ignored. This comment syntax works for input only, not for source code.

### Read input from stdin

Use `-s` to read input from standard input:

```bash
echo '1 2' | Nekomata -c '+' -s
```

### Run the same program on multiple inputs

Use `-m` to read multiple newline-separated inputs. This is especially useful for testing golf solutions against many cases.

### Show command-line help

```bash
Nekomata -h
```

## Using the REPL

If Nekomata is installed, start the REPL with:

```bash
Nekomata
```

Otherwise:

```bash
cabal run Nekomata
```

You will see a prompt like this:

```
Nekomata REPL - type \H for help
>>>
```

Enter code and press Enter to evaluate it:

```
>>> "Hello, World!"
Hello, World!
```

Type `\H` to see the available REPL commands.

### Enter input in the REPL

Use `\Input` to replace the current stack with new input:

```
>>> \Input 1 2
>>> +
3
```

If you run `\Input` with no arguments, the stack is cleared.

### Builtin names and tab completion

Every builtin has:

- a full ASCII name such as `\add`
- a one-character short form such as `+`

In the REPL, TAB completion can turn a full name into its short form. For example, typing `\add ` and then pressing TAB replaces it with `+`.

The trailing space matters, because several builtin names may share a prefix.

## Output Modes

Nekomata is non-deterministic, so a program can produce multiple results. Output modes control how those results are reported.

| Mode     | CLI Flag  | REPL Command   | Meaning                         |
| -------- | --------- | -------------- | ------------------------------- |
| `all`    | (default) | `\Mode all`    | Print all results               |
| `first`  | `-1`      | `\Mode first`  | Print only the first result     |
| `last`   | `-t`      | `\Mode last`   | Print only the last result      |
| `count`  | `-n`      | `\Mode count`  | Print the number of results     |
| `exists` | `-e`      | `\Mode exists` | Print whether any result exists |

### `all` (default)

Print every result, one per line.

Use `-l N` to limit how many results are shown:

```bash
Nekomata -c '1 2?1 2?+' -l 2
```

In the REPL, use `\Mode all` and `\Limit N`.

### `first`

Print the first result only:

```bash
Nekomata -c '1 2?1 2?+' -1
```

In the REPL: `\Mode first`.

### `last`

Print the last result only:

```bash
Nekomata -c '1 2?1 2?+' -t
```

In the REPL: `\Mode last`.

This mode was added in version 0.8.0.0, so older online environments may not support it.

### `count`

Print the number of possible results:

```bash
Nekomata -c '1 2?1 2?+' -n
```

In the REPL: `\Mode count`.

### `exists`

Print `True` if at least one result exists, otherwise `False`:

```bash
Nekomata -c '1 2?1 2?=' -e
```

In the REPL: `\Mode exists`.
