## What is Nekomata?

Nekomata is an experimental **non-deterministic concatenative golfing language** created by [AlephAlpha](https://github.com/AlephAlpha). It is designed for [code golf](https://codegolf.stackexchange.com/) — the art of writing programs in as few bytes as possible.

The language has three distinctive characteristics:

### 1. Concatenative (Stack-Based)

Like Forth or Joy, Nekomata programs are sequences of words that operate on a stack. Values are pushed onto the stack, and functions pop values and push results. Concatenating two programs runs them in sequence.

### 2. Non-Deterministic

Nekomata supports **non-deterministic programming** natively. A function can return multiple results simultaneously, and the interpreter explores all possible branches. This is especially powerful for golf challenges involving searching, partitioning, or constraint satisfaction.

### 3. Custom Code Page

Nekomata uses a custom 256-character encoding, where each byte corresponds to a single instruction. This allows many specialized operators to fit in a single byte, keeping programs compact.

## Key Features

- **Arbitrary-precision rational numbers** — no overflow or rounding issues
- **Lazy lists** — supports infinite lists and strings
- **Automatic vectorization** — many builtins work on lists element-wise
- **Flexible output modes** — show all results, first, last, count, or existence check
- **REPL** — interactive development with tab completion, mode switching, and stack inspection
