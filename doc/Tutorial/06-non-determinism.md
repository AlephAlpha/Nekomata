# Non-Deterministic Computation

Non-determinism is the most distinctive feature of Nekomata — and often the hardest to grasp. This chapter explains it step by step.

## What is Non-Determinism?

A non-deterministic computation has **multiple possible results** at the same time. This is not randomness — it is a deterministic exploration of all possibilities. The interpreter tracks every branch and outputs all successful results.

In Nekomata, non-determinism is the default. Even simple operations can branch into multiple execution paths.

## The Choice Function: `?`

The built-in `?` (`\choice`) pops two values and non-deterministically pushes one of them back.

```
1 2?
```

This program has two possible results: `1` or `2`. By default, the interpreter outputs all results, one per line:

```
1
2
```

## How Branches Combine

When you have multiple choice points, branches combine like a Cartesian product:

```
1 2?1 2?+
```

Here, two independent `?` calls each choose between `1` and `2`. The possible choices are:

- `1` + `1` = `2`
- `1` + `2` = `3`
- `2` + `1` = `3`
- `2` + `2` = `4`

The output is:

```
2
3
3
4
```

Note that `3` appears twice because there are two distinct paths that lead to it.

Now compare with:

```
1 2?:
```

Here, `?` chooses between `1` and `2`, then `:` duplicates. The possibilities are:

- Choose `1`: stack is `1 1` → result `2`
- Choose `2`: stack is `2 2` → result `4`

Output:

```
2
4
```

The key difference: in the first example, two independent choices produce four paths; in the second, one choice produces two paths.

## Duplicate Results

Non-determinism explores **paths**, not just values. Different paths can lead to the same value, and the interpreter counts them as separate results. This matters for the `count` and `all` modes.

## Failure

A computation may also have **no result**. This is called **failure**. The built-in `!` (`\fail`) always fails.

More usefully, many built-ins fail when a condition is not met. For example, `=` (`\eq`) returns the value if the top two values are equal, and fails otherwise.

```
1 2?2 3?=
```

Breaking this down:
1. `1 2?` chooses between `1` and `2`
2. `2 3?` chooses between `2` and `3`
3. `=` succeeds only if the two chosen values are equal

The only path that succeeds is choosing `2` from both choices:

```
2
```

All other paths fail.

This is how Nekomata handles conditions — it uses **failure as filtering**.

## Predicates Throughout the Language

Many builtins work as predicates that succeed or fail:

| Function | Succeeds if... |
|----------|---------------|
| `=` | Values are equal |
| `<` | First is less than second |
| `P` | Value is positive |
| `Q` | Value is prime |
| `N` | List is non-empty |

This pattern lets you write things like:

```
Ň           -> print all natural numbers (non-deterministic generator!)
ŇP          -> print all positive natural numbers
ŇQ          -> print all prime numbers
```

`Ň` (`\natural`) generates non-deterministic natural numbers (0, 1, 2, 3, ...). Adding `Q` filters to only primes.

## Non-Deterministic List Operations

Many list operations are non-deterministic:

| Function | Description |
|----------|-------------|
| `~` | Choose any element from a list |
| `S` | Choose any subset |
| `↕` | Choose any permutation |
| `s` | Choose any suffix |
| `p` | Choose any prefix |
| `q` | Choose any contiguous subsequence |
| `;` | Split list at any position |
| `ŧ` | Choose n-tuples from a list |

### Example: Anagrams

```
↕=
```

With input `"abc" "bca"`:
1. `↕` generates all permutations of `"abc"`
2. `=` checks if it matches `"bca"`

The result is `True` only if the permutation matches.

## Managing Non-Determinism

### Collecting Results: `a` (`\allValues`)

Collect all possible values of a non-deterministic computation into a list:

```
1 2?a    -> [1,2]
```

### Counting: `n` (`\countValues`)

Count how many results exist:

```
1 2?n    -> 2
```

### Unique Values: `ũ` (`\uniqueValue`)

Remove duplicate results:

```
1 2?1 2?+ũ   -> 2 3 4   (3 appears only once)
```

### First/Last Result

- `¡` (`\firstValue`): Get the first result
- `¤` (`\lastValue`): Get the last result

## Non-Deterministic Particles

Particles that relate to non-determinism:

### `ᶜ` — OrApply

Optionally apply a function 0 or 1 time:

```
5ᶜ←         -> 5 or 4
```

### `ᶦ` — Iterate

Apply a function 0 or more times, returning every intermediate result:

```
1:ᶦ{$ᵉ+    -> 1, 1, 2, 3, 5, 8, ...
```

### `ʰ` — On Any

Apply a function to any one element of a list:

```
[1,2,3]ʰ{→}  -> [2,2,3] or [1,3,3] or [1,2,4]
```

## The Big Picture

Non-determinism lets you write **search problems** declaratively:

1. Generate possibilities (e.g., `~`, `↕`, `Ň`, `S`, `?`)
2. Impose constraints (e.g., `=`, `<`, `P`, `Q`)
3. The interpreter finds all solutions

This is reminiscent of logic programming (Prolog) or constraint programming, but embedded in a concise stack-based syntax.

### Example: Sophie Germain Primes

Find Sophie Germain primes (primes `p` where `2p+1` is also prime):

```
Ƥ←½Q
```

- `Ƥ` generates a prime non-deterministically
- `←` decrements it
- `½` halves it (fails if not even)
- `Q` checks the result is prime

This is a 4-byte solution that finds all Sophie Germain primes.
