# Particles (Higher-Order Functions)

## What are Particles?

A **particle** is a higher-order function that modifies another function. The function being modified can be a built-in, a block, or even another particle-modified function.

Particles have short names that are always **Unicode modifier letters** (superscript characters), distinguishing them visually from built-in functions.

A particle is written **before** the function it modifies:

```
ᵐ{1+}    map the block {1+}
ᶠP       filter by isPositive
ʷ{←P}    while loop
```

## Mapping and Iterating

### `ᵐ` — Map

Apply a function to each element of a list.

```
[1,2,3]ᵐ{1+}    -> [2,3,4]
```

### `ᵚ` — Map With

Apply a binary function element-wise, using the top of stack as the second argument.

```
[1,2,3]4ᵚ+      -> [5,6,7]   same as [1,2,3]4+
```

### `ᶻ` — Zip With

Zip two lists together using a function. Fails if lengths don't match.

```
[1,2,3][4,5,6]ᶻ+   -> [5,7,9]
[1,2][3,4,5]ᶻ+     -> fails (length mismatch)
```

### `ᶾ` — Zip With (Truncating)

Zip two lists, truncating the longer one.

```
[1,2,3][4,5]ᶾ+     -> [5,7]
```

### `ᵒ` — Outer Product

Apply a function to every pair of elements from two lists (Cartesian product).

```
[1,2][3,4]ᵒ,       -> [[1,3],[1,4],[2,3],[2,4]]
```

### `ʲ` — Concat Map

Map a function that produces lists, then concatenate the results.

```
["ab","cd"]ʲ{…}    -> flattened result
```

## Filtering and Testing

### `ᶠ` — Filter

Keep elements where the function succeeds.

```
[1,2,0,3]ᶠP        -> [1,2,3]   (0 is not positive, so filtered out)
```

### `ᵖ` — Predicate

Succeed if the function succeeds when applied to the top value. The value is not consumed — it is passed through unchanged.

```
5ᵖP                 -> 5   (5 is positive)
0ᵖP                 -> fails
```

### `ᵗ` — Predicate Not

Succeed if the function **fails** when applied.

```
0ᵗZ                 -> 0   (0 is zero, so isNonzero fails, predicate succeeds)
5ᵗZ                 -> fails
```

## Control Flow

### `ᶜ` — Optionally Apply (OrApply)

Non-deterministically apply the function 0 or 1 time.

```
5ᶜ←                 -> 5 or 4
```

This creates two branches: one where nothing happens, one where decrement is applied.

### `ᶦ` — Iterate

Apply a function 0 or more times. Returns **all intermediate results** (non-deterministic).

```
1:ᶦ{$ᵉ+            -> 1, 1, 2, 3, 5, 8, ...  (Fibonacci!)
```

This starts with `1 1` on the stack (`1:`), then iterates a swap-and-add step. Each iteration produces the next Fibonacci number.

### `ʷ` — While

Apply a function repeatedly **until it fails**. Only the **last successful result** before failure is returned.

```
ʷ{←P}              -> keeps decrementing while positive, returns the last positive value
```

With input `5`: `5 → 4 → 3 → 2 → 1 → 0` (`P` fails on 0, so result is `1`).

### `ˡ` — Length While

Count how many times the function can be applied before it fails. Returns the count.

```
5ˡ{←P}             -> 4   (4 decrements before reaching 0)
```

This is useful for counting steps in algorithms.

### `ʸ` — Fixed Point

Apply a function **until the result stabilizes** (until successive applications produce equal values).

```
[1,[2,[3]]]ʸj      -> [1,2,3]   (flatten by repeatedly concatenating)
```

## Arity Modification

### `ᵃ` — On Both

Apply a function to the top two values on the stack.

```
1 2 ᵃ→             -> 2 3   (increment both)
```

### `ˣ` — No Pop

Apply a function without popping its arguments. The outputs are added on top of the stack.

```
1 ˣ{2+}            -> 1 3   (1 remains, 1+2=3 pushed)
```

### `ᵈ` — Dip

Save the top value, apply a function to the rest, then restore the saved value.

```
1 2 ᵈ{3+}          -> 4 1   (save 1, compute 2+3=4, restore 1)
```

### `ᵉ` — Dup Dip

Apply a function, then push the original top value back.

```
1 2 ᵉ+             -> 3 2   (1+2=3, then push original top 2)
```

Note the difference from `ᵈ`: `ᵈ` restores what was saved, while `ᵉ` copies the original top after applying the function.

## Other Particles

### `ʰ` — On Any

Non-deterministically apply a function to **one element** of a list.

```
[1,2,3]ʰ{→}        -> [2,2,3] or [1,3,3] or [1,2,4]
```

### `ʳ` — Fold Left

Fold a binary function over a list.

```
[1,2,3,4]ʳ+        -> 10   (((1+2)+3)+4)
[1,2,3,4]ʳ*        -> 24   (((1*2)*3)*4)
```

### `ᵏ` — First Integer

Find the smallest non-negative integer `n` such that the function succeeds when applied.

```
[1,2,4]ᵏf          -> 3   (3 is the first non-negative integer not in the list)
```

### `ᴷ` — Any Integer

Non-deterministically find any non-negative integer `n` such that the function succeeds.

```
[1,2,4]ᴷf          -> 0 or 3 or 5 or 6 or ...
```

## Particle Composition

Particles can be chained. For example, to map a filter over a list of lists:

```
[[1,0,2],[0,3]]ᵐᶠZ   -> [[1,2],[3]]
```

This maps (`ᵐ`) the filter-nonzero (`ᶠZ`) function over each sublist.
