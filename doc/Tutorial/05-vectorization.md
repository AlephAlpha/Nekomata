# Vectorization

> **Disclaimer:** This tutorial was generated by an AI and may contain inaccuracies. If you find any errors, please open an issue on the [Nekomata GitHub repository](https://github.com/AlephAlpha/Nekomata).

Many built-in functions in Nekomata are **automatically vectorized** over lists. This means they apply element-wise when given list arguments, reducing the need for explicit `ᵐ` (map) calls.

## Unary Functions

For unary functions, vectorization simply applies the function to each element:

```
[1,2,3]→                     -> [2,3,4]
["a","b"]H                   -> [97, 98]  (char to code point)
```

This is equivalent to `ᵐ{→}` and `ᵐ{H}`, but shorter.

## Binary Functions

Binary functions have two vectorization behaviors:

### Pad Vectorization (`+`, `-`)

Functions like `+` (`\add`) and `-` (`\sub`) use **pad vectorization**: if one list is shorter, it is padded with zeros.

```
[1,2,3]4+                    -> [5,6,7]       (scalar + list)
4[1,2,3]+                    -> [5,6,7]       (list + scalar)
[1,2,3][4,5]+                -> [5,7,3]       (pad shorter with 0)
[1,2][3,4,5]+                -> [4,6,5]       (pad [1,2] with 0: [1,2,0])
```

### Fail Vectorization (`*`, `/`)

Functions like `*` (`\mul`) and `/` (`\div`) use **fail vectorization**: the two lists must have the same length, otherwise the computation fails.

```
[1,2,3][4,5,6]*              -> [4,10,18]     (same length — OK)
[1,2,3][4,5]*                -> fails         (lengths differ)
[1,2,3]4*                    -> [4,8,12]      (scalar + list works)
```

## Practical Example: Identity Matrix

The canonical Nekomata solution for generating an identity matrix uses vectorization through outer product:

```
ᵒ-¬
```

With input `3`:
- `ᵒ-` computes the outer product of `[0,1,2]` with subtraction: `[[0,-1,-2],[1,0,-1],[2,1,0]]`
- `¬` (logical not) maps `0 -> 1` and non-zero -> `0`, giving `[[1,0,0],[0,1,0],[0,0,1]]`

## Vectorization vs. Explicit Map

While vectorization is automatic for builtins, you still need `ᵐ` for custom functions in blocks:

```
[1,2,3]ᵐ{1+}                 -> [2,3,4]     (explicit map of a block)
[1,2,3]1+                    -> [2,3,4]     (automatic vectorization)
```

Both produce the same result, but the second form is shorter when using a built-in.
