## The Stack Model

Nekomata is a **concatenative** language, which means programs work by manipulating a **stack**. Think of the stack as a pile of values where:

- Values are **pushed** onto the top
- Functions **pop** values from the top
- Results are **pushed** back onto the top

The stack is conceptually infinite. Input values are cycled repeatedly to fill the initial stack.

After a program finishes, the **top of the stack is printed** as the output.

### A First Example

```
"Hello, World!"
```

This pushes the string `"Hello, World!"` onto the stack. The output is `Hello, World!` (the quotes are omitted when a string is at the top).

### A Numeric Example

```
1 2+
```

Instructions are separated implicitly by context:
1. `1` pushes the integer `1`
2. `2` pushes the integer `2`
3. `+` pops the top two values (`2` and `1`), adds them, and pushes `3`

The stack becomes `3 ...`, so `3` is printed.

## Data Types

Nekomata has three fundamental data types: **numbers**, **chars**, and **lists**.

### Numbers

Numbers are **arbitrary-precision rational numbers**. They can be integers, fractions, or negative.

| Expression | Value | Notes |
|------------|-------|-------|
| `42` | `42` | Integer literal |
| `-42` | Cannot write directly | Use `42_` (negation) |
| `1/2` | Cannot write directly | Use `1\2` (fraction syntax) |
| `-3/4` | Cannot write directly | Use `3\4_` or `3_4/` |

Digits run together form a single number. Separate numbers with spaces:

```
1 2 3     -> pushes 1, 2, 3
123       -> pushes 123 (one number)
```

### Chars

Chars represent single characters from Nekomata's custom code page.

```
'a        -> pushes the char 'a'
```

A char is written as a single quote followed by the character. In output, chars are always shown with surrounding quotes:

```
>>> 'a
'a'
```

### Lists

Lists can contain any mix of data types, including nested lists:

```
[1, 'a', "Hello", [2, 3]]
```

Lists are **lazy**, meaning you can create infinite lists. However, the interpreter cannot output infinite lists directly — they can only be used as intermediate values.

### Strings

Nekomata does not have a separate string type. A **string is a list of chars**. When all elements of a list are chars, the interpreter prints it as a string (without quotes):

```
"Hello, World!"
```

is equivalent to:

```
['H', 'e', 'l', 'l', 'o', ',', ' ', 'W', 'o', 'r', 'l', 'd', '!']
```

An empty list and an empty string are the same value, written as `[]` or `""`.

## Basic Stack Operations

These built-in functions manipulate the stack directly:

| Short | Name | Arity | Description |
|-------|------|-------|-------------|
| `^` | `\drop` | `1 -> 0` | Remove top of stack |
| `:` | `\dup` | `1 -> 2` | Duplicate top |
| `$` | `\swap` | `2 -> 2` | Swap top two: `a b -> b a` |
| `§` | `\rot3` | `3 -> 3` | Rotate three: `a b c -> c a b` |
| `v` | `\over` | `2 -> 3` | Copy second to top: `a b -> b a b` |

### Examples

```
1 2 3 ^    -> stack: 1 2    (drop 3)
1 2 $      -> stack: 2 1    (swap)
1 :        -> stack: 1 1    (dup)
1 2 3 §    -> stack: 3 1 2  (rot3)
1 2 v      -> stack: 1 2 1  (over)
```
